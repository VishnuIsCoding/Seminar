01. What is a Graph?

A Graph is a non-linear data structure consisting of a finite set of vertices (also called nodes) connected by edges. Unlike a Tree, which enforces a strict parent-child hierarchy (one-to-many), a Graph allows many-to-many relationships between nodes — making it the most general and powerful data organization model.

Formally, a graph G is defined as an ordered pair:

G = (V, E)
  • V (Vertices / Nodes): A finite, non-empty set of objects. In a social network, each user is a vertex. In a road map, each intersection is a vertex.
  • E (Edges / Arcs): A set of pairs of vertices representing connections. In a social network, a friendship link is an edge. In a road map, a road segment is an edge.

Key Distinction from Trees: In a Tree structure, the relationship between parent and children is one-to-many. In a Graph, the relationship is many-to-many — any vertex can connect to any other vertex, and cycles are permitted.

Real-World Applications: Computer networks (routing), social networks (friend suggestions), web page indexing (Google PageRank), dependency resolution in compilers, GPS navigation systems, and neural network architectures.

02. Graph Abstract Data Type

The Graph ADT defines what operations can be performed on a graph without specifying how the data is stored internally. This abstraction allows developers to swap between adjacency matrix and adjacency list implementations without changing the code that uses the graph.

GraphADT.hpp — Complete Interface Definition
template <typename T>
class GraphADT {
public:
    // ── Structural Modification ──
    virtual void addVertex(const T& vertex) = 0;
    virtual void removeVertex(const T& vertex) = 0;
    virtual void addEdge(const T& src, const T& dest, int weight = 1) = 0;
    virtual void removeEdge(const T& src, const T& dest) = 0;

    // ── Query & Inspection ──
    virtual bool isAdjacent(const T& u, const T& v) const = 0;
    virtual int getDegree(const T& vertex) const = 0;
    virtual int getInDegree(const T& vertex) const = 0;   // For digraphs
    virtual int getOutDegree(const T& vertex) const = 0;  // For digraphs
    virtual std::vector<T> getNeighbors(const T& vertex) const = 0;

    // ── Traversal ──
    virtual std::vector<T> getVertices() const = 0;
    virtual std::vector<std::pair<T,T>> getEdges() const = 0;

    // ── Status ──
    virtual bool isEmpty() const = 0;
    virtual int vertexCount() const = 0;
    virtual int edgeCount() const = 0;
    virtual void clear() = 0;

    // ── Virtual Destructor ──
    virtual ~GraphADT() = default;
};

03. ADT Operations in Detail

Structural Operations

These operations modify the shape and size of the graph by adding or removing vertices and edges.

  • addVertex(v) — Insert a new node into the graph. Allocates space in the internal data structure.
  • removeVertex(v) — Delete a node and all edges incident to it.
  • addEdge(u, v) — Create a connection between two existing vertices.
  • removeEdge(u, v) — Delete the connection between two vertices.

Query & Inspection Operations

These operations examine the current state of the graph without modifying it.

  • isAdjacent(u, v) — Returns true if there is an edge from u to v.
  • getDegree(v) — Returns the number of edges connected to vertex v.
  • getNeighbors(v) — Returns a list of all vertices directly connected to v.
  • getInDegree(v) / getOutDegree(v) — For directed graphs, returns the number of incoming / outgoing edges.

04. Directed Graph (Digraph)

In a Directed Graph (Digraph), every edge has a specific direction. An edge is an ordered pair (u, v), meaning there is a path from vertex u to vertex v, but not necessarily from v back to u.

Key Property: (u, v) ≠ (v, u). The relationship is asymmetric.

Applications: Web page links (A links to B but B may not link to A), task dependencies (A must finish before B), social media followers (A follows B but B may not follow A).

DirectedGraph.cpp
class DirectedGraph {
    int V;
    std::vector<std::list<int>> adj;

public:
    DirectedGraph(int vertices) : V(vertices), adj(vertices) {}

    void addEdge(int u, int v) {
        // Only add in ONE direction (u → v)
        adj[u].push_back(v);
        std::cout << "Edge added: " << u << " → " << v << std::endl;
    }

    int getInDegree(int v) {
        int count = 0;
        for (int i = 0; i < V; i++) {
            for (int neighbor : adj[i]) {
                if (neighbor == v) count++;
            }
        }
        return count;
    }

    int getOutDegree(int v) {
        return adj[v].size();
    }
};

05. Undirected Graph

In an Undirected Graph, edges have no direction — the connection is mutual and symmetric. If vertex A is connected to vertex B, then B is automatically connected to A.

Key Property: (u, v) = (v, u). Every edge is bidirectional.

Applications: Social network friendships (mutual), physical cables in a LAN, chemical bonds between atoms, collaboration networks.

UndirectedGraph.cpp
class UndirectedGraph {
    int V;
    std::vector<std::list<int>> adj;

public:
    UndirectedGraph(int vertices) : V(vertices), adj(vertices) {}

    void addEdge(int u, int v) {
        // Add in BOTH directions (bidirectional)
        adj[u].push_back(v);
        adj[v].push_back(u);
        std::cout << "Link: " << u << " ↔ " << v << std::endl;
    }

    int getDegree(int v) {
        return adj[v].size();
    }

    void printNeighbors(int v) {
        std::cout << "Neighbors of " << v << ": ";
        for (int n : adj[v]) {
            std::cout << n << " ";
        }
        std::cout << std::endl;
    }
};

06. Weighted Graph

A Weighted Graph assigns a numerical value (called a weight or cost) to each edge. This value typically represents distance, travel time, bandwidth, or monetary cost.

Key Property: Each edge (u, v) carries a third parameter w — the weight.

Applications: GPS navigation (shortest distance), airline route optimization (cheapest fare), network routing (minimum latency), Dijkstra's and Prim's algorithms.

WeightedGraph.cpp
struct Edge {
    int destination;
    int weight;
};

class WeightedGraph {
    int V;
    std::vector<std::list<Edge>> adj;

public:
    WeightedGraph(int vertices) : V(vertices), adj(vertices) {}

    void addEdge(int src, int dest, int weight) {
        adj[src].push_back({dest, weight});
        // For undirected weighted:
        // adj[dest].push_back({src, weight});
        std::cout << src << " --(" << weight << ")--> " << dest << std::endl;
    }

    int getEdgeWeight(int u, int v) {
        for (const Edge& e : adj[u]) {
            if (e.destination == v) return e.weight;
        }
        return -1; // Edge not found
    }
};

07. Adjacency Matrix Representation

The Adjacency Matrix is a 2D array of size V × V. The cell M[i][j] stores 1 if an edge exists between vertex i and vertex j, otherwise 0. For weighted graphs, the cell stores the edge weight instead of 1.

  • Space: O(V²) — requires a full V×V grid regardless of edge count.
  • Edge Lookup: O(1) — direct array access.
  • Find Neighbors: O(V) — must scan entire row.
  • Best For: Dense graphs where E ≈ V².
AdjacencyMatrix.cpp
class AdjacencyMatrix {
    int V;
    int** matrix;

public:
    AdjacencyMatrix(int numVertices) : V(numVertices) {
        matrix = new int*[V];
        for (int i = 0; i < V; i++) {
            matrix[i] = new int[V]();  // Zero-initialize
        }
    }

    void addEdge(int u, int v) {
        matrix[u][v] = 1;
        matrix[v][u] = 1;  // Remove for directed
    }

    bool hasEdge(int u, int v) {
        return matrix[u][v] == 1;
    }

    void printMatrix() {
        for (int i = 0; i < V; i++) {
            for (int j = 0; j < V; j++) {
                std::cout << matrix[i][j] << " ";
            }
            std::cout << std::endl;
        }
    }

    ~AdjacencyMatrix() {
        for (int i = 0; i < V; i++) delete[] matrix[i];
        delete[] matrix;
    }
};

08. Adjacency List Representation

The Adjacency List uses an array of linked lists (or vectors). Each index i in the array corresponds to vertex i, and the linked list at that index contains all vertices adjacent to i.

  • Space: O(V + E) — stores only existing edges, highly memory-efficient.
  • Edge Lookup: O(V) worst case — must traverse the list.
  • Find Neighbors: O(degree) — extremely fast neighbor traversal.
  • Best For: Sparse graphs (most real-world networks).
AdjacencyList.cpp
class AdjacencyList {
    int V;
    std::vector<std::list<int>> adj;

public:
    AdjacencyList(int numVertices) : V(numVertices), adj(numVertices) {}

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);  // Remove for directed
    }

    bool hasEdge(int u, int v) {
        for (int neighbor : adj[u]) {
            if (neighbor == v) return true;
        }
        return false;
    }

    void printList() {
        for (int i = 0; i < V; i++) {
            std::cout << i << ": ";
            for (int neighbor : adj[i]) {
                std::cout << neighbor << " → ";
            }
            std::cout << "NULL" << std::endl;
        }
    }
};

09. Complexity Analysis

Choosing between Adjacency Matrix and Adjacency List depends on the density of the graph and which operations are performed most frequently. Below is a detailed comparison.

Operation Adjacency Matrix Adjacency List When to Prefer
Space O(V²) O(V + E) List for sparse, Matrix for dense
Add Vertex O(V²) — rebuild O(1) List for dynamic growth
Add Edge O(1) O(1) Both efficient
Check Edge (u,v) O(1) O(V) Matrix for frequent lookups
Find All Neighbors O(V) O(degree) List for BFS/DFS traversals
Remove Edge O(1) O(V) Matrix for frequent removals

Summary & Key Takeaways

Graph = G(V, E)

The most versatile non-linear data structure, capable of modeling any many-to-many relationship in real-world systems.

ADT ≠ Implementation

The Graph ADT separates interface from storage. We can swap Matrix for List without changing application logic.

Choose Wisely

Dense graphs → Matrix (O(1) lookups). Sparse graphs → List (O(V+E) memory). Most real-world graphs are sparse.
